《地理信息科学-A》课程笔记。

课程核心知识点

1. 空间参考与坐标系统

基本概念

  1. 地理坐标系统（geocs）：是地球表面空间要素的定位参照系统，由经度和纬度进行定义。经度是从本初子午线开始向东或详细度量角度，纬度是从赤道平面向北或向南量度角度。

  2. 投影坐标系统：基于投影的平面坐标系统。（分类）

  3. 几何变换：利用一系列控制点和转换方程式在投影坐标上配准数字化地图、卫星图像或航空照片的过程。

椭球体和参考椭球（大地基准）之间的关系

投影坐标系统是基于某种地理坐标系进行投影而来，同一种地理坐标系统可以按照不同的投影方法得到不同的投影坐标系统，基于不同的地理坐标系统进行相同投影也将得到不同的投影坐标系统。

geocs与procs之间的联系

地理信息系统：是由计算机硬件、软件和不同的方法组成的系统，该系统设计支持空间数据的采集、管理、处理、分析、建模和显示，以便解决复杂的规划和管理问题。GIS脱胎于地图学，是计算机科学、地理学、测绘遥感学、环境科学、城市科学、空间科学、信息科学和管理科学等众多学科交叉融合而成的新兴学科。但是，地理信息系统与这学科和系统之间既有联系又有区别:

（1）GIS 与机助制图系统。机助制图是地理信息系统得主要技术基础，它涉及GIS中的空间数据采集、表示、处理、可视化甚至空间数据的管理。地理信息系统和数字制图系统的主要区别在于空间分析方面。一个功能完善的地理信息系统可以包含数字制图系统的所有功能，此外它还应具有丰富的空间分析功能。

（2）GIS与DBMS（数据库管理系统）。GIS除需要功能强大的空间数据的管理功能之外，还需要具有图形数据的采集、空间数据的可视化和空间分析等功能。因此，GIS 在硬件和软件方面均比一般事务数据库更加复杂，在功能上也比后者要多地多。

（3）GIS与CAD系统。二者虽然都有参考系统，都能描述图形，但 CAD 系统只处理规则的几何图形、属性库功能弱，更缺乏分析和判断能力。

（4）GIS与遥感图像处理的系统。遥感图像处理的系统是专门用于对遥感图像数据处理进行分析处理的软件。它主要强调对遥感栅格数据的几何处理、灰度处理和专题信息提取。 这种系统一般缺少实体的空间关系描述，难以进行某一实体的属性查询和空间关系查询以及网络分析等功能。

如果两幅地图的投影相同，但是叠置时仍然存在一定程度的偏移，请问如何实现两幅图之间的配准

可以利用几何变换实现两幅地图的配准，首先选择一定的几何变换方法来构建两个坐标系之间的关系，然后根据该方法对于控制点的数目要求，在两幅地图上选择数量足够的同名控制点进行几何变换，从而实现两幅地图之间的配准。

请问地面控制点（ground control point）的定义与作用

从图像到地图变换使用的控制点，其的图像坐标和真实世界坐标都能识别。

从理论上来说，地面控制点应该选取单一的、明显的像元形式显示出来的要素，是具有明显的用于识别地物位置的特征点，如：道路交叉点、桥梁、建筑物拐角等，而且这些的点坐标已知或可以间接获得。可以使用地面控制点，来对变换方程的未知参数进行估算，从而实现图像到地图的变换。

控制点最少数量：对于多项式变换而言，若多项式为$m$次，则最小需要$(m+1)(m+2)/2$个控制点。

2. 空间数据模型

基本概念

矢量数据模型

栅格数据模型

矢量数据模型:矢量数据模型用带有坐标的点线面等矢量数据对具有精确位置和边界的空间要素进行表述，矢量数据模型表达空间要素时，位置明确而属性隐含，数据量小，但结构复杂，且进行空间分析的难度较大。矢量数据模型可以分为无拓扑矢量数据模型：如shapefile，有拓扑矢量数据模型：如coverage，和基于对象的矢量数据模型：如geodatabase（有拓扑）。

栅格数据模型：栅格数据模型利用规则格网对于地学现象的连续分布特征进行表述，实质上是对连续空间分布现象的离散化，采用了一个二维矩阵对某一种现象进行描述，像元值表示该位置上的空间现象特征、使用行号与列号来确定每个象元点或面元（cell）的空间位置。

请问arcgis shape和arcgis coverage，哪种文件格式是含有拓扑关系

Coverage文件格式含有拓扑关系，包含的拓扑关系有

  1. 连接性：弧段之间通过结点彼此链接。

  2. 面定义：由一系列相连的弧段定义面。

  3. 邻接性：弧段有方向性，且有左多边形和右多边形。

请问九交模型包括哪8种空间关系

九交模型描述了Equal分离，Disjoint分离，Intersects分离，Touch分离，Overlap重叠，Cross交叉，Within包含于，Contains包含八种空间关系。

3. 空间数据编辑

矢量数据编辑的常见错误

  1. 定位错误:空间要素的几何位置错误，如空间要素缺失和其他与空间要素几何位置错误有关的错误。

  2. 拓扑错误：与空间要素之间的逻辑不一致错误，诸如:悬挂结点（过伸、未及），伪结点、弧段在结束结点未正确与其他弧段接合，弧段的方向错误，如河流的起点的海拔低于终点的海拔、多个线图层的要素重叠（铁路与高速公路）、多边形未闭合、相邻多边形存在间隙、多边形之间重叠，多边形包含多个标记点、两个多边形图层的外部边界线未重合等，拓扑错误是基于某些拓扑关系而言，我们将与某些拓扑关系不符的情况视为拓扑错误。如在行政图种，要求多边形之间都要相邻，如存在不相邻的多边形就存在拓扑错误。

请问容差（tolerance）在数据编辑过程中的作用

可以通过设置容差可以让软件自动接合距离小于该指定容差的点，线和多边形轮廓，这种方式在自动处理小的过伸、未及和重复线上非常有用，从而大大减少了数据编辑的工作量。但容差不应该设置的过大，大的容差会改变线和多边形的形状。对于无法自动处理的拓扑错误，在手工编辑时，可以通过设置适合的容差，让鼠标在目标点附近时，自动选中准确的位置点。

线的平滑与化简

线的泛化：通过消除线条上的某些点而简化线条的过程，douglas-Peucker算法缺点：简化后的线条具有明显的角。

线的平滑：通过对线条添加新节点而使其平滑的过程，这些新节点往往由样条等数学函数产生。

样条函数：采用分段光滑的处理方式，每段采用三次函数，各段的连接处保持光滑，由样条函数生成的平滑曲线通过已知点，且曲线的一阶和二阶的导数连续。

4. 属性数据管理

数据库表格规范化的作用

规范化：将一个包括所有属性的数据表格分解成小的表格，同时保持它们之间必要的连接的过程。

作用：

  1. 避免表中多余数据浪费数据库空间，以及避免可能导致的数据完整性问题。

  2. 确保独立表格中属性数据可以被单独维护、更新，并在需要时可以将各表格链接起来。

  3. 有利于形成分布式数据库。

数据库中主键与外键的定义

主键：表中能唯一确定一条记录的一个或者多个属性。

外键：表中参考另一个表的主键的一个或者多个属性，其值与另一个表中的主键值相对应，从而在进行关联和合并时能将两个表联系起来。

表格之间关系的四种基本类型

表格之间通常包含四种关系类型：一对一，一对多，多对一，多对多。

一对一：表示目标表中的一条记录与源表中一条记录关联。

一对多：目标表中的一条记录与源表中多条记录关联。

多对一：目标表中的多条记录与源表中一条记录关联。

多对多：目标表中的多条记录与源表中多条记录关联。

合并和关联的定义

合并：源表和目标表通过主键和外键合并在一起，仅适合 一对一关系或多对一关系的表格。

关联：源表和目标表通过主键和外键连接在一起，表之间依然独立，仅仅建立记录之间的联系。源表和目标表的四种关系都适用。

5. 数据显示与制图

地图的基本要素

地图的基本要素包含图名，图例，指北针，比例尺，地图主体，文字说明（注记）、图廓等。地图中还可以包含格网、投影信息，数据质量信息，图幅编号，图幅系列，插图，野外调查时间和制图时间等，这些要素共同将地图信息传递给读者。

地图符号

地图符号是地图的语言，是在地图上表达空间对象的图形记号，它通过尺寸、形状和颜色来表示事物 空间的位置、形状、质量和数量特征，是表达地理 现象与发展的基本手段。

点状符号通常较规则；图形位置固定（不随位置变化）； 符号大小与地图比例尺无关， 但有确定的定位点和方向。 如居民点、气象站等。可以设置点状符号形状(shape)、 大小(size)、角度(angle)、 颜色等。

线状符号是定位于几何上的线， 符号沿某个方向延伸，宽度与 地图比例尺无关，长度与地图 比例尺相关，具有定位线； 线 状符号常可进一步分解成具有 单一特征的线状符号。如河流、 公路、航线等。设置参数： 线形(type)、线宽 (width)、结点形状(cap)、顶点连接方式(join)、颜色。

面状符号是定位于几何上的面， 它具有封闭的边界线；通常在封闭区内配置点状符号、阴影线 或颜色。如森林、土地等。设置参数: 填充类型、前景色、 轮廓线宽度与颜色。

地图注记

地图上起说明作用的文字和数字的总称，是地图内容的重要组成部分。制图者往往把注记当作一种地图符号，因为注记也有多种变化类型，如字体的变化（大小，字形，字样等）、注记放置位置的变化、大小是否随比例尺变化而变化。

图层数据的分组或分类

数据分类的五种方法：对数据进行分层或分类，然后不同类别的数据进行配色方案、符

号配置等制图处理。

等间隔：使得分类结果中每个类的数据变化范围或数值间距相等。

等频率，即利用分位数对数据进行分类，将数据等分为N类，每类中所含有的的样本数相等。

标准离差：将平均值向上或向下偏移标准值（0.5,1.0等）的单元为类别（或组）的分类点。

自然断点：分组原则是将组内差异最小化，组间差异最大化 。

几何分级法：几何间隔使得每组元素的平方和最小化。

定量地图的6中常见类型：

点描法地图：用同一的点状符号表示空间数据，每一个符号代表的一个单位值。

等值区域地图：用分级颜色的方案表示基于行政单元的派生数据的变化特征。

分级符号地图：用大小不同的符号（圆圈、方形、三角形）来代表数据的不同数值范围。

饼状统计地图：饼状统计地图是分级圆圈符号图的一种变种，可以显示两套定量数据：圆圈的大小依据数值的大小比例而定，而圆内的分区可以用来显示数值的构成。

流量地图：以不同宽度的线状符号来显示不同数量的流量数据的地图；

等值线地图：用等值线系统来表示一个表面的地图。

地图的排版设计

地图设计：为达一定目标而进行的视觉设计。地图设计的目的是为了增强地图，使其易于理解和传递正确消息和信息。

排版：又称平面组织，是对地图的不同要素进行排列和组合。排版关注的主要方面包括焦点、顺序和平衡问题

视觉层次：是指将三维效果或深度引入可视平面的开发过程

6. 矢量数据分析

缓冲区

缓冲区（buffer）是围绕空间要素一定宽度（缓冲距离）的区域，用于分析空间要素与周围要素之间的空间邻近性。；建立缓冲区可以把空间分为两个区域：一个区域位于空间要素的指定距离之内，另一个区域位于空间要素的指定距离之外。在指定距离之内的区域叫做缓冲区。在建立缓冲区时，缓冲距离可以设置为常数，也可以根据给定字段取值而变化；一个空间要素可以按照一定距离生成一个缓冲区，也可以按照不同距离建立多个缓冲区；线性要素可以在线的两侧生成缓冲区，也可以在一侧生成缓冲区等。

缓冲分析（proximity analysis）

缓冲分析（proximity analysis）缓冲区分析是邻近度分析的一种，指以点、线、面实体为基础，建立其周围一定距离范围内的缓冲区图层，然后将生成的缓冲区与其他图层进行叠加运算，进一步分析得到所需结果。在建立缓冲区时，缓冲距离可以设置为常数，也可以根据给定字段取值而变化；一个空间要素可以按照一定距离生成一个缓冲区，也可以按照不同距离建立多个缓冲区；线性要素可以在线的两侧生成缓冲区，也可以在一侧生成缓冲区等；还可以建立缓冲区分析模型用于度量在主体对象的邻近区域内，邻近对象受主体对象影响的变化。按照主体对邻近对象作用强度的不同，可以分为线性模型，二次模型，指数模型等。

叠置分析（Overlay - union， intersection， identify）

叠置分析（Overlay - union,intersection,identify）：叠置分析是将两个具有相同坐标系统的要素图层的几何形状和属性组合在一起，生成新的输出土图层的GIS分析操作。叠置分析的常用四种叠置分析方法为：求交（intersection)，联合(Union），对称差异（symmetrical Difference),和识别（Identity）。

联合：保留输入图层和叠置图层全部要素的一种"多边形与多边形叠置"方法。

求交：仅保留输入图层和叠置图层共同区域范围要素的一种叠置分析方法，也是叠置分析的首先方法。

对称差异：仅保留输入图层和叠置图层各自独有的区域范围的要素的一种叠置分析方法。

识别：仅保留落在输入图层定义的区域范围内要素的一种叠置分析方法。

碎屑多边形：一种地图叠置的常见错位，即沿着两个输入图层的共同边界出现的面积很小的多边形。

距离的量测问题

距离量测是指要素之间直线（欧氏）距离的量测。量测可以在一个图层中的点到另一个图层的点之间进行，输出结果为一个图层中的所有点到另一个图层中所有点的欧氏距离。也可以在一个图层中的各个点到另一个图层中与该点最邻近的点或线之间进行，输出结果为输入图层的各个点到邻近图层最邻近要素的距离和角度。

雷普利函数（Ripley K函数）的定义和用途

用于辨别点集数据在一定的距离范围下是随机分布、聚集分布还是均匀分布的一种空间统计量。

优势在于可用于探讨在不同的空间尺度下，点数据的空间分布模式，也可以用于寻找点簇的聚集范围。

基于 Ripley's K 函数的多距离空间聚类分析工具是另外一种分析事件点数据的空间模式的方法。该方法不同于此工具集中其他方法（空间自相关和热点分析）的特征是可对一定距离范围内的空间相关性（要素聚类或要素扩散）进行汇总。在许多要素模式分析研究中，都需要选择适当的分析比例。例如，该分析通常需要距离范围或距离阈值。在多个距离和空间比例下研究空间模式时，模式会发生变化，而这通常可反映对运行中的特定空间过程的控制。Ripley's K 函数可表明要素质心的空间聚集或空间扩散在邻域大小发生变化时是如何变化的。

8. 栅格数据分析

基本概念

局域运算（local）：以像元为基本运算单元的栅格运算，是栅格分析的核心。局域运算由单个或者多个输入栅格生成一个新的栅格，新栅格的像元值由输入栅格对应的像元值计算得到。

邻域运算（focal）：邻域运算也称为焦点运算，涉及到一个焦点像元和一组环绕像元。邻域运算要求一个输入栅格，输出栅格的像元值由以输入栅格对应像元为焦点的邻域内的像元值计算得到。

分区运算（zonal）：

分区：具有相同值的像元分组，分区可以是连续的也可以是不连续的。分区运算可对一个或两个输入栅格图层进行处理。 若一个输入图层，则计算每个分区的几何特征(面积、厚度)；若两个输入图层，则做分区内的统计。

某个旅游景点位于较为陡峭的山区，需要从山脚下到山顶安装一条索道，请问如何找出备选的安装区域

假设索道修建为直线，要求为沿各山脚处找一个点，修建到山顶的索道，该索道的成本需要在1000万以下，且乘坐该索道能尽量欣赏到景区内优美的风景。假设索道的修建成本仅与修建距离相关，成本为1000元一米，乘坐站台（山顶上的站台高度为常数值，山脚处的站台高度可变，以便于缆车不会与山体发生碰撞）的修建成本仅与高度相关，成本为2000元一米。

  1. 根据自然距离量算，量算出各山脚处到山顶的欧式距离。

  2. 根据视线操作原理，在要求各山脚处站台与山顶处站台的连线要高于沿线任意一点的山体高度的条件下，求出各山脚处站台的最小高度，以及各山脚处站台与山顶站台之间的高差

  3. 由上面计算得到的水平距离和高程差，计算在不同山脚处设置索道时，各索道的实际长度

  4. 计算在不同山脚处安装索道的总成本（站台成本加上索道成本），将建设总成本小于1000万的索道作为备选索道

  5. 对整个景区的景观进行分类，按照风景的优美程度，设置为1-5，5个级别，5为最优美区域

  6. 对于不同索道进行视域分析，获取不同索道的的视域栅格。

  7. 利用不同索道的视域栅格对于景观分类栅格进行分区统计（统计视域内景观分类图层的像素值的总和），从而获取乘坐不同索道能看到景观的总计得分。

  8. 选取得分做高的索道作为最终安装索道

9. 地形分析

坡度、坡向、曲率、TWI，…等地形因子

坡度：地表位置上高度变化率的度量。

坡向：斜坡方向的度量。

TWI（Topographic Wetness Index ：地形湿度指数）：TWI是表示流域内水流和滞水空间分布的指标；

twi = log( accumulation/tan(slope) )，计算流量累计栅格时使用的是多流向算法。

TPI（Topographic Position Index：地形位置指数）：在给定半径范围内内，一个像元的高程与周围像元的平均高程之差，TPI正值表示一个焦点像元的高程比其窗口内的高程均值大，凸地形；负值则小，凹地形；接近零，则平坦地形。

TRI（Terrain Roughness Index：地形粗糙指数）：中心像元与邻近像元的高程差的绝对值的平均值， TRI通常用3x3窗口计算，反映了地形的局部起伏的差异性。

HPI（hydrographic position index :水文位置指数）：在dem中突出地表输水地貌的位置。

Average Slope：平均坡度是根据移动窗口内中心享元年和邻近像元之间的高程差和距离来计算的。正、负或零表示表面的主要凸、凹或平坦形状。

RPI（Relative Position Index）：相对位置指数，给出了一个像元在水平上离最近的山脊或溪流有多远的近似估计。

UPstream：上游小流域到本像元的最大距离。

Downstream：某像元到该像元中水汇入河网中的像元的距离。

Relative elevation：流域内所有像元值减去流域出口的像元值（高程）。

DEM和TIN都可以用来表达地面高程模型，请问二者的区别及其可能的应用

数字高程模型(DEM，Digital Elevation Model)基于高程点的规则分布，通常用规则的格网表示地表各处的高程值，是一种栅格数据模型。

Tin是局域高程点的不规则分布，用一系列不重叠的三角形来近似模拟陆地表面，从而构成不规则的三角网，是一种矢量数据模型。

TIN 与DEM可以相互转换。

相对与DEM而言，Tin可以利用较少的数据量对地形进行更为精确的表达。构建TIN时，往往在地形较为平坦地区采用较少的采样点以减少数据量，而在地形变化较大的区域采用更多的采样点以更好的描述地形的起伏。而DEM（高程栅格）对于平坦区域和地形变化较大区域都采用规则格网进行表达，往往数据量较大，且在分辨率较低时，无法很好表示复杂地形。

与DEM相比，TIN的更新更为灵活，如可以向构建好的TIN中添加断线以定义地表的不连续处，或直接添加新的高程采样点来对复杂地区进行三角网的加密，以提高地形表达的准确率。而DEM的像元大小是固定的，无法通过增加部分区域的采样点对其进行更新，要想提高DEM对地形表达的准确率，唯一的办法只能提高DEM的分辨率，而这需要重新对整个区域的高程数据进行编辑，往往耗费较高。

与TIN相比，DEM在地形分析上的计算效率更高。DEM（高程栅格）的简单数据结构使得比较容易在其上进行邻域分析，因此利用高程栅格计算坡度、坡向、表面曲率、TWI等地形因子更加快捷。而TIN的数据结构较为复杂，在进行地形分析时计算量较大，且随着三角形数目的增加，计算量迅速增加。

因此往往利用TIN进行地形制图和三维显示，以更准确的对地形进行表示，而往往利用高程栅格进行地形分析，以提高计算效率。

10. 地形制图与分析

地图制图的几种基本方法
  1. 等高线法：等高线表示的是高程值相等的点的连线，按照一定的等高距绘制出等高线可以对研究区域的地形进行表示。
  2. 垂直剖面法：利用垂直剖面来表示沿诸如远足小道、道路或者河流等线条上的高度变化，从而对地形进行表示。
  3. 地貌晕渲图（阴影地形图）：通过模拟太阳光与地表要素相互作用下的地形容貌来对地形进行表示。
  4. 分层设色法：通过用不同的颜色符号表示不同的高程分区（范围）来对地形进行表示。
  5. 透视图：通过生成在一定观察点所看到的陆地表面三维视图的方法来对地形进行表示。
地形因子 – 高程、坡度、坡向、曲率、地形湿度因子、…

高程指的是某点沿铅垂线方向到绝对基面的距离，称绝对高程，简称高程。某点沿铅垂线方向到某假定水准基面的距离，称假定高程。

影响坡度和坡向量算的因子。

  1. 用于生成坡度和坡向的DEM数据的空间分辨率。一个高分辨率DEM生成的坡度和坡向图层，应当比低分辨率DEM所生成的包含更多细节。

  2. DEM的质量。

  3. 坡度和坡向算法。

  4. 局部地形。陡坡地区坡度估算误差较大，平坦地区坡向估算误差较大。

表面曲率：用于确定某一个像元位置的表面是向上凸还是向下凹的统计量，是地形表面凹凸程度的度量，可分为剖面曲率和平面曲率。剖面曲率是沿最大坡度方向的曲线的曲率，平面曲率是沿垂直与最大坡度方向的曲线的曲率。

11. 视域与流域分析

视域分析 - 可视性

视域：从一个或者多个观察点可以看见的地表范围。

视域分析：提取视域的过程。

基础：视线操作，根据DEM，判断观察点与观察目标之间的可视性，如果沿视线有任意一点处的地表高于视线，则该目标对该观察点不可视。

流域分析 – 填洼fill，流向，汇流，河网，子流域提取

流域：具有共同出水口的地表水流经的集水区域

流域分析：利用DEM进行水流方向计算，汇流面积计算，流域边界的提取和河网的分析

基于区域的流域勾绘：将研究区域划分成一系列流域，每个流域对应一个河段

基于点的流域勾绘：用所选的点生成流域

填洼：洼地是指无法确定流向的像元或在空间上连接的像元组，这往往是中心像元的高程值低于周围像元的高程值的结果。填洼是指在DEM中对洼地的高程值进行填充从而消除洼地的过程。一种常用方法是将洼地的高程值填充到其周围像元的最低高程值来完成填洼过程

为什么填洼：DEM中的洼地往往被认为是由错误数据引起的，例如：数据采样影响，或者将浮点型高程数据舍零 取整为整数型等，这类错误必须被消除，而对于实际存在的小洼地，在进行流域分析时也应该进行消除，这是因为洼地代表了一个内排水区域，在存在洼地的DEM上进行水流方向计算和汇流面积计算时会出现错误，进而导致流域和河网提取的错误，这是与实际情况不符的。而在实际情况中，一些洼地在降雨时也会被雨水填充，流经被雨水填充后的洼地的水流也会流向其他地点，这也表明我们需要在DEM中对这种真实存在的洼地进行填充。

流向：水流离开已填洼DEM中每一个像元时的方向。计算流向可以使用单流向算法和多流向算法。

单流向算法：水流离开某一像元时只会朝着一个方向流动，常用的算法为D8算法，该算法以8邻域（3*3的窗口）为基础，将中心像元的流向指向为与中心像元距离权重坡度最大的像元。

多流向算法：水流离开某一像元时可朝着多个方向流动，并可以计算出朝着每个方向流动的水流的比例。

汇流面积（流量累计）：某个像元的汇流面积表明有多少个上游像元的水流经该像元，流量累计栅格可以根据流向栅格得到。

河网：具有一定排水量的区域。给定汇流面积栅格和阈值来对河网进行提取；

子流域提取：子流域表示每一个河段（link)所对应的流域，为每个河段提取子流域成为子流域提取。一个河段对应的子流域可以被定义为流入该河段的所有像元，可以根据流向栅格计算得到，而一个河段对应的子流域可以通过合并组成该河段的像元所对应流域得到。

单流向与多流向

使用 D8、SFD（单流向），多流向 (MFD) 或 D-Infinity (DINF) 方法创建从每个像元到其下坡相邻点的流向的栅格。

D8 流向法可对每个像元到其最陡下坡邻域的流向进行建模。

指定计算流向时使用的流向法的类型。

D8 —根据 D8 流向法分配流向。此方法可将流向分配至最陡下坡相邻点。这是默认设置。

MFD —根据 MFD 流向法分配流向。此方法可将多流向分配至所有下坡邻域。

DINF —根据 D-Infinity 流向法，使用三角面的最陡坡度指定流向。

为什么划分流域？

流域包含了相对完整的地理过程

降水、入渗、汇流、蒸散、植被过程 ……

流域是相对独立的空间单元

相对封闭，受到外界的干扰较少（垂向、横向物质能量交互）

流域具有较明确的物质运动途径

水分主要在重力作用下沿地表运动，易于观测

12. 空间插值

空间插值：用已知点的数值来估算其他点的数值的过程。

根据有限个已知点的数值，通过距离或者其它空间关系，推测出其它位置的近似值的方法，就叫做空间插值。

趋势面插值与回归插值

趋势面插值：用数学函数（多项式方程）定义的曲面对所有已知样本点进行最佳拟合，并利用拟合后的多项式方程计算未知点的值。

趋势插值法特点：可获得感兴趣区域渐进趋势的平滑表面。生成的表面几乎不能穿过各原始数据点，因为对整个表面执行的是最佳拟合。插值结果的值域可能被外推。插值表面对异常值（极高值和极低值）非常敏感，尤其是在表面的边缘处。

适用情况：

  1. 表面在各位置间出现趋势性渐变时，可将该表面与采样点拟合，例如，工业区的污染情况；

  2. 检查或排除长期趋势或全局趋势的影响。

可通过描述物理过程的低阶多项式创建渐变表面，例如，污染情况和风向。但使用的多项式越复杂，为其赋予物理意义就越困难；

插值表面对异常值（极高值和极低值）非常敏感，尤其是在表面的边缘处。

回归插值：

全局回归插值：在整个研究区内，建立目标变量与多个自变量之间的回归方程，根据已知点对于回归方程的参数进行估算，再将回归方程用于估算每个未知点处的目标变量值

全局回归的优缺点：

优点：

  1. 操作简单，结合邻域运算、栅格运算、Sample、SPSS等即可实现;

  2. 利用了丰富的地理协变量信息；

  3. 表达简单，各变量贡献明确；

  4. 模型较稳定，异常值少。

缺点：

  1. 实际的预测变量和地理协变量之间的关系可能随时间和空间变化；

  2. 已知点代表性不足时，回归方程可能存在偏差；

  3. 地理协变量出现极值或异常值时，或超出建模时的值域，会导致预测结果异常。

进行全局多项式的局限性：一阶可拟合平面，二阶可拟合包含一个弯曲的表面，三阶可拟合包含两个弯曲的表面。当表面具有多种形状（如延绵起伏的地表）时，单个全局多项式将无法很好地拟合

局部多项式：对未知点进行估值时，仅使用其邻域范围内的已知点对多项式方程的参数进行估算，再将多项式用于未知点的估值。

线性回归分析的弱点：

  1. 自变量和因变量之间必须满足线性关系。

  2. 存在多重共线性，自相关性和异方差性。

  3. 对异常值非常敏感，异常值会严重影响回归线和最终的预测值。

  4. 多重共线性会增加系数估计的方差，并且使得估计对模型中的微小变化非常敏感，结果是系数估计不稳定。

  5. 在多个自变量的情况下，我们可以采用正向选择、向后消除和逐步选择的方法来选择最重要的自变量。

地理加权回归：全局模型（Global model）在分析之前，就假定了因变量与自变量之间的

关系具有区域同质性（Homogeneity)，而在实际情况中，变量间的关系可能因地理位置的变化而发生变化（空间非平稳性），而GWR作为一种局部的空间回归方法，可以用来探测空间关系的非平稳性。GRW在仅利用未知点邻域内的已知点来估算回归方程的参数，且在估算过程中各已知点的权重不同，权重由衰减函数计算且往往距离未知点越近的已知点的权重越大

泰森多边形：将整个研究区域进行多边形划分，每个多边形内的任意点与多边形内的已知点的距离最近，然后将泰森多边形内已知点的赋予多边形内的所有点（边界内无变化，边界处突变）（值不会外推）

自然邻域法：在泰森多边形基础上，把待插值点也作为一个中心求出空间中距离该待插值点最近的区域，再在该区域中求出各个多边形所占据的面积，按面积比例对周围涉及的样本加权来进行插值（该表面将通过输入样本且在除输入样本位置之外的其他所有位置平滑变化），对于未知点的估值不会超过已知值的范围（值不会外推）

密度曲面、IDW、曲面样条函数

IDW（反距离加权、精确插值）：假设未知值的点受近距离已知点的影响比远距离已知点的影响更大，因此对周围样点按距离赋予不同的权重，近的权重大，远的权重小。q=0为移动平均，q=1为线性插值，q-\>inf，退化为泰森多边形。

IDW特点：

  1. 幂值q通常取0.5~3，默认值为2，大的幂值将更加强调最近点；

  2. 可通过使用的周围点的个数或搜索的半径，限制所使用的输入点；

  3. 输入点对插值点的影响是各向同性的；

  4. 预测值仅限于用于插值的样点值的范围。因此，如果未对极值采样，便无法创建山脊或山谷。

  5. 结果有"牛眼"效应。

  6. 对输入数据的统计属性没有明确假设；非常适合用于超大型输入数据集。

样条函数：精确插值方法，生成一个通过已知点的表面，且该表面必须拥有最小的曲率（每个点的二阶导数平方和最小）；且必须确保表面平滑（连续且可微分，即一阶导数表面连续）

克里金插值（半变异函数、普通克里金插值法）

一种随机性插值方法（可以预测值的误差进行评估），可对区域化变量进行线性无偏最优估计，是地统计学的主要内容之一。克里格插值法与反距离加权法有些类似，两者都通过对已知样点赋权重来求得未知点的值。

不同的是，在赋权重时：反距离加权法只考虑已知样点与未知样点的距离远近；克里格方法不仅考虑距离，还通过对已知样点值的分析，统计并拟合出样点值间的差异与其距离远近之间的关系，再施加线性、无偏、最优等约束条件，联立求解已知点对插值点的权重，已知点的权重之间会相互影响。

克里格法是一个多步骤的过程，包括数据的探索性统计分析、变异函数建模、解方程确定权重、创建插值面，还包括研究方差表面等；

克里金方法的特点：（ppt15-56)

无偏和估计方差最小

屏蔽效应

兼顾各向异性

可显示估计方差

权值可为负值

无偏：区域化变量在各点上估计量的数学期望等于其在同一位置上的真实值。

最优：是指区域化变量在各点上估计量与其在同一位置上的真实值的方差最小。

半变异函数：用于描述样点数据的差别与距离之间关系，可以用于测定区域化变量的空间自相关性。

块金：距离为0时的半变异，表示量测及分析误差或微小变异。

变程：半变异开始稳定时的样对距离，表示的是区域化变量的空间自相关的范围。

基台值：半变异图中达到恒定的半变异称为总基台值，总基台值与块金之差为基台值。

普通克里金插值法：假定区域化变量在空间上不存在趋势，变量值的空间变异仅受到空间自相关因素的影响的一种克里金方法。

插值结果的验证（独立验证与交叉验证）

独立验证（Independent Validation; Hold-out）：一部分样点建模，另一部分验证，相互独立；建模点不参与验证，验证点不参与建模。

交叉验证：在样本量较少时的一种结果验证方法，往往需要进行多次验证，每次的验证过程为独立验证，取多次验证结果的平均值作为交叉验证结果。常见的交叉验证方法包括K折验证（将样本平均分成K份，每次利用K-1份样本进行训练，另外一份进行验证，重复K次）和留P验证（留出p个样点验证，剩下的n-p样点训练，重复C(n, p)次）。

13. 最小耗费距离与网络分析

阻抗

阻抗：一种耗费、成本或是代价，链路阻抗是穿过链路所需要的成本。转弯阻抗是在节点处从一个链路到另一个链路所需要的耗费，通常为时间。

网络

网络是按一定拓扑结构连接在一起的多个节点和链路的集合，是承载物质、能量和信息流动的一维受限空间。

形式化定义：从某种相同类型的实际问题中抽象出来、用数学中的图论来表达的一种模型，即由节点和连线构成，表示诸多对象及其相互联系的图。

网络分析的实质是通过对网络状态的模拟和分析，研究资源在网络上的流动和分配；同时，对网络结构及资源等的优化问题进行研究。

链路link

链路link：道路网络中由两个节点所确定的路段，也称为边。

节点：链路的一个交会点。

最短路径分析：在网络中寻找节点见累计阻抗最小的路径。

迪杰斯特拉最短路径

一种最短路径的计算方法，基本思想是：

1.指定起点s，引进两个集合U和S，U记录已求出最短路径的顶点以及相应的最短路径长度，S记录还未求出最短路径的顶点以及该顶点到起点s的最短距离；

2.初始时，U中只有起点s，S中是除s之外的顶点；

3.从S中找出到s最近的顶点，并将其加入到U中，更新S中的顶点和顶点对应的最短距离。

4.重复3的操作，直到遍历完所有顶点。

旅行商问题：寻找从一个起点出发，通过且仅通过每个给定点一次，最后再回到起点的成本最小路径。

车辆路径问题（多货车送货问题）：针对不同地理位置上多个客户的货物需求，为一个车队规划适当的行车路线，在特定约束下使得成本最低。

空间可达性定义： 指某一地点到达其他地点、或其他地点到达这一地点的便利程度，是一种对点与线、点与点地理要素进行空间关系分析的过程。

Costpath成本距离

成本（耗费）：穿过每个像元所需要的成本。

成本距离量测：可以生成多个输出结果。分别为最小累积耗费栅格，该栅格输出每个像元到最近源像元的最小累积成本；方向栅格输出每个像元到最近源像元的方向；配置栅格显示每个像元到哪个源像元的成本最低；最小耗费路径输出像元到源像元的最小耗费路径。

Arcgis中成本距离量测是基于结点-链接表示法，结点代表像元的中心，链接（包括横向链接和对角线链接）连接结点和邻接像元。横向链接的距离为1个像元，以横向链接的方式从一个像元到其相邻像元的耗费距离为两个像元耗费值的平均值，以对角线链接的方式从一个像元到其相邻像元的耗费距离为两个像元耗费值平均值的1.414倍。

由于成本距离量算是基于结点-链接表示法，所以每个像元到源像元有多条路径，每条路径的累计成本不同，所以要根据某种最短耗费路径算法来计算其到源像元的最小累计成本路径，并输出每个像元到源像元的最小累计成本。

资源配置

也称为定位与分配（Location-Allocation）问题，研究如何实现最佳服务。

定位问题是指依据资源需求点的分布，确定最优的供应点配置，以实现最佳的服务效率。

而分配问题则是根据供应点配置状态，将资源需求点分配到合适的供应点使得请求服务的效率最高。

15. 地理编码与动态分段

地理编码：指将描述性的详细地址或地名转换为地理坐标（如经纬度)的过程，一个有效的地址应该是独一无二的。

逆地理编码：将地理坐标转换为地址信息的过程，通常返回附近的POI (point of interest) 、AOI(area of interest)信息反向地理编码提供了坐标定位引擎，帮助用户通过地面某个地物的坐标值来反向查询得到该地物所在的行政区划、所处街道、以及最匹配的标准地址信息。

地理编码的要素

必须明确需要编码的地理对象，地理对象不同则实现的方法不同;

必须有确定的参考系统，可以是基于坐标的或是基于地理标识的地址，就是一种建立在地理标识参考系下，使用自然语言描述地理位置的参考系统;

必须有唯一的编码规则。

地理编码过程：预处理，匹配，地图显示
  1. 预处理：包含解析和标准化

解析：把一个地址分解为许多的组成分

标准化：对于不含完整组成分的地址，标准化可以鉴别并按照顺序排列每一个地址组成份

  1. 匹配：在对一个地址进行预处理后，需要将其与参考数据库中大量表示地址并包含位置属性（有经纬度信息）的文字进行相似性匹配，在这些表示地址的文字中匹配到相似度最高的文字，最后，返回匹配结果和相应的经纬度信息做为这个过程的成果

  2. 标绘（地图显示）。

地理编码与Join和relate的异同

相同点：两者都涉及到值的匹配的过程。Join和relate通过对两个表中主键和外键的值进行匹配将两个表中的记录关联起来，地理编码也需要将预处理后的文体地址与参考数据库中的大量的文体地址进行匹配，从而返回地理坐标。

区别：

两者的功能不同：地理编码将描述性的详细地址转换成地理坐标，join和relate将不同表的记录之间关联起来。

两者的匹配过程不一样：join和relate进行值的匹配时，主键的值和外键的值必须完全相同才算匹配成功。而地理编码的匹配是基于相似度的匹配，两者的值不需要完全相同，通过计算两者的相似度，将参考数据库中相似度最高或相似度大于某一阈值的地址所对应地理坐标返回作为结果。

反向地理编码和Identify的异同

反向地理编码：将地理坐标转换成描述性的详细地址的过程。

Identify：获取选中要素的相关属性值。

相同点：两个都可以获取与某一位置相关的描述性信息。

不同点：Identify获取的是选中要素的属性值，以点图层为例，如果要获取一个点的相关属性值，必须要准确选中该点，如果选择的是该点旁边的位置，则无法获取任何信息。而反向地理编码将要查询的地理坐标与参考数据库中地理坐标进行相似匹配，返回相似度最高的地理坐标所对应的地址，所以在输入地理坐标与参考数据库中的地理坐标无法准确匹配时，往往返回的时输入坐标周围的某个POI或是AOI。

动态分段

动态分段：沿着一条路径动态计算事件位置的过程。

路径：线性量测系统与其几何特性一起存储的线要素。

事件：沿路径发生的线性参考数据（属性）。

是对事件表中所存储事件在线状要素上的地图位置 （形状）进行计算的过程。

通过动态分段可将多组属性与线状要素的任意部分相关联。

1. 基本概念解释

数据
  • 数据是指客观事物的属性、数量、位置及其相互关系等的符号描述。
  • 在空间数据库中，数据可以是一个数，如某一点的高程值、一个多边形的面积等，也可以是一组符号组成的字符串，如一个地名、一个河流注记、一幅图像。
  • 空间数据不同于普通的数据，它具有空间性、时间性、多维性和大数据量等特点。而且数据之间不仅有传统的关联关系，更多地还有空间关系，这就给数据的处理和利用带来了更多的难度。
信息

信息 是向人们或机器提供关于现实世界新的事实的知识，是数据、消息中所包含的意义，它不随载体的物理设备形式的改变而改变。

特征有客观性、适用性、可传输性和共享性等。

数据与信息

数据和信息之间是相互联系的。数据是反映客观事物属性的记录，是信息的具体表现形式。数据经过加工处理之后，就成为信息；而信息需要经过数字化转变成数据才能存储和传输。

数据和信息是有区别的。从信息论的观点来看，描述信源的数据是信息和数据冗余之和，即：数据=信息+数据冗余。

数据是数据采集时提供的，信息是从采集的数据中获取的有用信息。

信息可以简单地理解为数据中包含的有用的内容。

也可以说数据在未被接收对象获取前可以称为数据，一旦被对象获取，即可称为信息。

空间
  • 空间（space）是一个应用很广泛的名词，空间是客观存在的物质空间，空间是—个复杂的概念，具有多义性，既有与时间对应的含义，也有"宇宙空间"的含义。
  • 日常语义上的"空间"是指事物之间的距离或间隔。
  • 从地理学的意义上讲，空间是人类赖以生存的地球表层具有一定厚度的连续空间域，是一个定义在地球表层空间实体集上的关系。GIS领域的space是指地理环境或地球表层空间，是地理信息系统表达和研究的对象。为了在GIS中对地理空间进行描述，常常需要借助于抽象的数学空间表达方法。
地理空间
  • 地球表面上的一切地理现象、地理事件、地理效应、地理过程统统都发生在以地理空间为背景的基础之上。
  • 地理空间（Geospace）是一个相对空间，是一个空间实体组合排列集（这些空间实体具有精确的空间位置），强调宏观的空间分布和空间实体间的相关关系（关系以各单个地理空间实体为联结的结点或载体）。

GIS中的数据来源和数据类型繁多，概括起来主要有以下几种来源:

⑴地图数据。来源于各种类型的普通地图和专题地图，这些地图的内容丰富，图上实体间的空间关系直观，实体的类别或属性清晰，实测地形图还具有很高的精度，是地理信息的 主要载体，同时也是地理信息系统最重要的信息源。

⑶地形数据。来源于地形等高线图的数字化，已建立的数字高程模型（DEM）和其他实测的地形数据等。

⑷属性数据。来源于各类调查报告、实测数据、文献资料、解译信息等。

⑸元数据。来源于由各类纯数据通过调查、推理、分析和总结得到的有关数据的数据，例如数据来源、数据权属、数据产生的时间、数据精度、数据分辨率、源数据比例尺、数据 转换方法等。

3. 解释栅格数据和属性数据之间的不同

矢量数据（Vector data）使用点和点的(x， y)坐标表示空间要素（点、线和面）。

栅格数据（Raster data）使用格网和格网单元表示要素的空间变化。

栅格数据结构

栅格结构是以规则的阵列来表示空间地物或现象分布的数据组织，组织中的每个数据表示地物或现象的非几何属性特征。

栅格结构的显著特点：属性明显，定位隐含，即数据直接记录属性的指针或数据本身，而所在位置则根据行列号转换为相应的坐标。

栅格数据的编码方法：直接栅格编码，就是将栅格数据看作一个数据矩阵，逐行（或逐列）逐个记录代码；

压缩编码，包括链码（弗里曼链码）比较适合存储图形数据；

游程长度编码通过记录行或列上相邻若干属性相同点的代码来实现；

块码是有成长度编码扩展到二维的情况，采用方形区域为记录单元；

四叉树编码是最有效的栅格数据压缩编码方法之一，还能提高图形操作效率，具有可变的分辨率。

矢量数据结构

矢量数据结构是通过记录坐标的方式尽可能精确地表示点、线和多边形等地理实体，坐标空间设为连续，允许任意位置、长度和面积的精确定义。

矢量结构的显著特点：定位明显，属性隐含。

矢量数据的编码方法：

对于点实体和线实体，直接记录空间信息和属性信息；

对于多边形地物，有坐标序列法、树状索引编码法和拓扑结构编码法。坐标序列法是由多边形边界的(x， y)坐标对集合及说明信息组成，是最简单的一种多边形矢量编码法，文件结构简单，但多边形边界被存储两次产生数据冗余，而且缺少邻域信息；树状索引编码法是将所有边界点进行数字化，顺序存储坐标对，由点索引与边界线号相联系，以线索引与各多边形相联系，形成树状索引结构，消除了相邻多边形边界数据冗余问题；拓扑结构编码法是通过建立一个完整的拓扑关系结构，彻底解决邻域和岛状信息处理问题的方法，但增加了算法的复杂性和数据库的大小。

矢量栅格数据的比较

矢量数据的优缺点：

优点为数据结构紧凑、冗余度低，有利于网络和检索分析，图形显示质量好、精度高；

缺点为数据结构复杂，多边形叠加分析比较困难。

栅格数据的优缺点：

优点为数据结构简单，便于空间分析和地表模拟，现势性较强；

缺点为数据量大，投影转换比较复杂。

两者比较：

栅格数据操作总的来说容易实现，矢量数据操作则比较复杂；

栅格结构是矢量结构在某种程度上的一种近似，对于同一地物达到于矢量数据相同的精度需要更大量的数据；在坐标位置搜索、计算多边形形状面积等方面栅格结构更为有效，而且易于遥感相结合，易于信息共享；矢量结构对于拓扑关系的搜索则更为高效，网络信息只有用矢量才能完全描述，而且精度较高。对于地理信息系统软件来说，两者共存，各自发挥优势是十分有效的。

1. 地图投影的重要性

地图投影有几个优点

首先，地图投影使得我们可以使用二维纸质或数字地图，而不是一个地球仪。

其次，地图投影使得我们可以使用平面或投影坐标，而不是经纬度。地理坐标的计算相对平面坐标而言更加复杂。

2. 根据所保留的性质描述地图投影的4种类型

正形投影（Conformal Projection）可以保持局部角度和形状。

等积投影（Equivalent Projection）可以保持区域面积准确大小。

等距投影（Equidistant Projection）可以保持在 某些线 上的比例尺一致性。

(等)方位投影（Azimuthal Projection）可以保持某些方向的准确性。

3. 通过投影或可展曲面描述地图投影的3种类型

等角投影：等角投影保留局部形状。要保留描述空间关系的各个角，等角投影必须在地图上显示以 90 度角相交的垂直经纬网线。地图投影通过保持所有角不变来加以实现。缺点是由一些弧线围起来的区域将在此过程中发生巨大变形。地图投影无法保留较大区域的形状。

等积投影：保留所显示要素的面积。为此，形状、角和比例等其他属性将发生变形。在等积投影中，经线和纬线可能不垂直相交。有些情况下，尤其是较小区域的地图，形状不会明显变形，且很难区分等积投影和等角投影，除非加以说明或进行测量。

等距投影：保留某些点间的距离。任何投影都无法在整幅地图中正确保持比例不变。不过，多数情况下，地图上总会存在一条或多条这样的线：比例沿着这些线将正确地保持不变。多数等距投影都具有一条或多条这样的线：在此类线中，地图上线的长度（按地图比例尺计算）与地球上同一条线的长度相同，无论它是大圆还是小圆，是直线还是曲线。此类距离被视为真实距离。例如，在正弦投影中，赤道和所有纬线就是其真实长度。在其他等距投影中，赤道和所有经线具有真实长度。而其他投影（例如，两点等距离）仍会显示地图上一点或两点与相隔点间的真实比例。请记住，任何投影都不能实现地图上的所有点是等距离的。

按投影面与地球的相对位置关系分为正轴投影、斜轴投影和横轴投影等；

按照投影面的形状分为圆锥投影、圆柱投影和方位投影等；

按投影面和地球的空间逻辑关系可以氛围相切和相割两类投影。

我国的地图投影中1：100万地形图采用了Lamber（兰勃特）投影（正轴等角割圆锥投影），大于1：100万的基本比例尺地形图采用高斯—克吕格投影（横轴等角切圆柱投影）。

第3章 矢量数据类型

论述点、线、多边形数据之间的叠加分析的内容和方法。

（1）点与多边形叠加

点与多边形叠加，实际上是计算多边形对点的包含关系，进行点是否在一个多边形中的空间关系判断。在完成点与多边形的几何关系计算后，还要进行属性信息处理。最简单的方式是将多边形属性信息叠加到其中的点上。当然也可以将点的属性叠加到多边形上，用于标识该多边形，如果有多个点分布在一个多边形内的情形时，则要采用一些特殊规则，如将点的数目或各点属性的总和等信息叠加到多边形上。通过点与多边形叠加，可以计算出每个多边形类型里有多少个点，不但要区分点是否在多边形内，还要描述在多边形内部的点的属性信息。通常不直接产生新数据层面，只是把属性信息叠加到原图层中，然后通过属性查询间接获得点与多边形叠加的需要信息。例如一个中国政区图（多边形）和一个全国矿产分布图（点），二者经叠加分析后，并且将政区图多边形有关的属性信息加到矿产的属性数据表中，然后通过属性查询，可以查询指定省有多少种矿产，产量有多少；而且可以查询，指定类型的矿产在哪些省里有分布等信息。

（2）线与多边形叠加

线与多边形的叠加，是比较线上坐标与多边形坐标的关系，判断线是否落在多边形内。计算过程通常是计算线与多边形的交点，只要相交，就产生一个结点，将原线打断成一条条弧段，并将原线和多边形的属性信息一起赋给新弧段。叠加的结果产生了一个新的数据层面，每条线被它穿过的多边形打断成新弧段图层，同时产生一个相应的属性数据表记录原线和多边形的属性信息。根据叠加的结果可以确定每条弧段落在哪个多边形内，可以查询指定多边形内指定线穿过的长度。如果线状图层为河流，叠加的结果是多边形将穿过它的所有河流打断成弧段，可以查询任意多边形内的河流长度，进而计算它的河流密度等；如果线状图层为道路网，叠加的结果可以得到每个多边形内的道路网密度，内部的交通流量，进入、离开各个多边形的交通量，相邻多边形之间的相互交通量。

（3）多边形叠加

多边形叠加是GIS最常用的功能之一。多边形叠加将两个或多个多边形图层进行叠加产生一个新多边形图层的操作，其结果将原来多边形要素分割成新要素，新要素综合了原来两层或多层的属性。叠加过程可分为几何求交过程和属性分配过程两步。几何求交过程首先求出所有多边形边界线的交点，再根据这些交点重新进行多边形拓扑运算，对新生成的拓扑多边形图层的每个对象赋一多边形唯一标识码，同时生成一个与新多边形对象一一对应的属性表。多边形叠加结果通常把一个多边形分割成多个多边形，属性分配过程最典型的方法是将输入图层对象的属性拷贝到新对象的属性表中，或把输入图层对象的标识作为外键，直接关联到输入图层的属性表。这种属性分配方法的理论假设是多边形对象内属性是均质的，将它们分割后，属性不变。也可以结合多种统计方法为新多边形赋属性值。多边形叠加完成后，根据新图层的属性表可以查询原图层的属性信息，新生成的图层和其它图层一样可以进行各种空间分析和查询操作。根据叠加结果最后欲保留空间特征的不同要求，一般的GIS软件都提供了三种类型的多边形叠加操作：并、叠和、交。

拓扑

拓扑关系：研究在拓扑变换下能够保持不变的性质，其描述了两个对象之间的关系，这种关系称之为拓扑关系。

拓扑邻接：表示图形中同类元素之间的拓扑关系。

拓扑关联：表示空间图形中不同类元素之间的拓扑关系。

拓扑包含：表示空间图形中，面状实体所包含的其他面状实体或线状、点状实体的关系。

线的维数为1，除了位置，还有长度属性。

面的维数为2，除了位置，还有面积和周长属性。

2. 拓扑在GIS中的重要性

  1. it ensures data quality， such as lines meeting perfectly and polygons closed properly.

  2. topology can enhance certain types of GIS analysis such as geocoding and traffic volume analysis.

  3. topological relationships between spatial features allow GIS users to perform spatial data query.

  4. 它保证了数据质量，如线能够交合和多边形能够封闭。

  5. 拓扑可以增强某些类型的GIS分析，如地理编码和交通流量分析。

  6. 空间特征之间的拓扑关系允许GIS用户执行空间数据查询。

  7. 根据拓扑关系，不需要利用坐标或者计算距离，就可以确定一种地理实体相对于另一种地理实体的空间位置关系。

  8. 利用拓扑数据有利于空间要素的查询。

  9. 可以利用拓扑数据作为工具，重建地理实体。

3. Shapefiles的主要优点

Shapefiles have two main advantages over topology-based data such as coverages.

  1. shapefiles can display more rapidly on the computer monitor.

  2. shapefiles are nonproprietary and interoperable， meaning that they can be used across different software packages.

与基于拓扑结构的数据（如coverages）相比，shapefiles有两个主要优势。

  1. shapefiles可以在计算机显示器上更迅速地显示。

  2. shapefiles是非专有的、可互操作的，也就是说，它们可以在不同的软件包中使用。

4. 解释地理关系数据模型和基于对象的数据模型之间的区别

两方面区别：

  1. 基于对象数据模型（object-based data model）将空间要素（Spatial Feature）的几何数据和属性数据存储于单一系统，而不是分离系统。

  2. 基于对象数据模型允许空间要素（对象）与一组属性和方法相关联。

5. 就空间要素的几何表示，阐述Geodatabase与Coverage之间的区别

地理数据库（Geodatabase） 在简单特征的几何表示方面类似于Coverage模型。

二者的主要区别在于regions 和 routes 两个合成特征：地理数据库不支持region 子类，且用带m（measure）值的polyline 代替 routes 子类。

Coverage和Shapefile是地理关系数据模型，它利用分离的系统来存储空间数据和属性数据，而Geodatabase是基于对象数据模型，它把空间数据和属性数据存储在唯一的系统中。

　　Coverage是拓扑的，Shapefile是非拓扑的。

　　Coverage支持三种基本拓扑关系：连接性、面定义、邻接性。

　　Shapefile多边形对于共享边界实际上有重复弧段且可彼此重叠，不同于Coverage所用的多个文件，它用几何学性质存储两个基本文件：以.shp为扩展名的文件存储要素几何学特征；以.shx为扩展名的文件保留要素几何特征的空间索引。

　　Shapefile:一种基于文件方式存储GIS数据的文件格式。至少由.shp，.dbf，.shx三个文件作成，分别存储空间，属性和前两者的关系。是GIS中比较通用的一种数据格式。

　　Coverage:一种拓扑数据结构，一般的GIS原理书中都有它的原理论述。数据结构复杂，属性缺省存储在Info表中。目前ArcGIS中仍然有一些分析操作只能基于这种数据格式进行操作。

Geodatabase: ArcInfo发展到ArcGIS时候推出的一种数据格式，一种基于RDBMS存储的数据格式，其有两大类：

  1. PersonalGeodatabse 用来存储小数据量数据，存储在Access的mdb格式中。

  2. ArcSDE Geodatabse存储大型数据，存储在大型数据库中Oracle， Sql Server， DB2等。可以实现并发操作，不过需要单独的用户许可。

6. 解释Geodatabase，要素数据集和要素类之间的关系

Geodatabase、要素数据集（Feature Dataset）和要素类（Feature Class）构成层次结构。

一个Geodatabase可以包含一个或多个要素数据集；一个要素数据集可以包含一个或多个共享同一坐标系统和区域范围的要素数据类。（无论在要素数据集中的还是独立的要素类，同一个Geodatabase中的要素类不可重名。）

7. 解释个人Geodatabase和文件Geodatabase之间的区别

Personal Geodatabase 将数据存储为文件拓展名为.mdb的Microsoft Access 数据库中，存储为table。

File Geodatabase 将数据存储在许多小文件中，这些小文件则位于以.gdb为拓展名的文件夹中，数据库大小无限制，可跨平台工作（如 Windows 和 Linux）。

第4章 栅格数据类型

数据处理

对采集的各种数据，按照不同的方式方法对数据形式进行编辑运算，清楚数据冗余，弥补数据缺失，形成符合用户要求的数据文件格式。

空间内插

通过已知点数据推求同一区域其他未知点数据的计算方法。

空间外推

通过已知区域数据，推求其他区域数据的方法。

空间数据压缩

从空间坐标数据集合中抽取一个子集，使这个子集在规定的精度范围内最好地逼近原集合，而又取得尽可能大的压缩比。

元数据

元数据是关于数据的描述性数据信息，它应尽可能多地反应数据集自身的特征规律，以便于用户对数据集的准确、高效与充分的开发与利用。元数据的内容包括对数据集的描述、对数据质量的描述、对数据处理信息的说明、对数据转换方法的描述、对数据库的更新、集成等的说明。

栅格数据

栅格结构是最简单的空间数据结构，是指将地球表面划分为大小均匀紧密相邻的网格阵列，每个网格作为一个像元或像素由行、列定义，并包含一个代码表示该像素的属性或量值，或仅仅包含指向其属性记录的指针。因此，栅格结构是以规则的阵列来表示空间地物或现象分布的数据组织，组织中的每个数据表示地物现象的非几何特征。

栅格数据的主要编码方式

  1. 直接栅格编码。最简单直观而又非常重要的一种栅格结构编码方法，就是将栅格数据看作一个数据矩阵，逐行（或逐列）逐个记录编码，可以每行都从左到右逐个像元记录，也可以奇数行地从左到右而偶数行的从右向左记录，为了特定目的还可采用其他特殊的顺序。

特点：编码简单，信息无压缩，无损失，但数据量大。

  1. 压缩编码方法。目的是用尽可能少的数据量记录尽可能多的信息，其类型又有信息无损编码和信息有损编码之分。

特点：所占存储空间小，易于检索、叠加、合并操作，编码效率提高，（有损编码）但有损原始数据。

（1） 链码。可以有效地压缩栅格数据，且对于估算面积、长度、转折方向的凹凸程度等运算十分方便，比较适合于存储图形数据。

（2） 游程长度编码。栅格图像常常有行（或列）方向上相邻的若干点具有相同的属性编码，因而可采取某种方法压缩记录重复内容。一种方法是只在某行（列）数据的编码以及相同的编码重复的个数；另一种方法是逐个记录各行（列）编码发生变化的位置和相应编码。

（3） 分块编码。块码是游程长度编码扩展到二维的情况，采用方形区域作为记录单元，每个记录单元包括相邻的若干栅格，数据结构由初始位置（行、列号）和半径，再加上记录单位的编码组成。

（4） 四叉树。将整个图像逐步分解为一系列被单一类型区域内含的方形区域，最小的方形区域为一个栅格像元。只要划分到仅代表一种地物或符合既定要求的少数几种地物时，则不再继续划分。

（5） 细分网格法编码。

游程编码是栅格数据的一种比较简单的保真压缩编码方法。编码中，将栅格阵列中具有相同属性值的邻近单元合并在一起，构成一个游程。

每个游程使用一对数字来表示(A， P)。其中A代表属性值或属性值的指针，P表示该游程最右端栅格的列号。（注：还有一种游程编码P是行程的长度即重复单元的个数。）

游程编码又称行程编码。

行程编码：栅格数据的一种压缩编码方式，是将行或列中重复的元素进行合并，以达到减少存储和数据冗余的目的，包括一维行程和二维行程。

数字地形模型（DTM）

Digital Terrain Model。是地形表面形态属性信息的数字表达，是带有空间位置特征和地形属性特征的数字描述。数字地形模型中，地形属性为高程时称为数字高程模型（DEM）。

TIN不规则三角网

TIN即不规则三角网，是一种表示数字高程模型的方法。TIN模型根据区域有限个点集，将区域划分为相连的三角面网络，区域中任意点落在三角面的顶点、边上或三角形内。如果点不在顶点上，该点的高程值通常通过线性插值的方法得到。

TIN又叫做不规则格网的DEM，是基于矢量的数字地理数据的一种形式，通过将一系列折点组成的三角形来构建。各折点通过由一系列边进行连接，最终形成一个三角网。在ArcGIS中主要有三种生成TIN的方式，即由矢量数据生成TIN、由栅格数据生成TIN和由Terrain数据集生成TIN。

格网DEM分析的主要应用

  1. 地形曲面拟合。DEM最基础的应用是求DEM范围内任意点的高程，在此基础上进行地形属性分析。由于已知有限个个网点的高程，可以利用这些格网点高程你和一个地形曲面，推求区域内任意点的高程。

  2. 立体透视图。绘制透视立体图是DEM的一个极其重要的应用。透视立体图能更好地反映地形的立体形态，非常直观。人们可以根据不同的需要，对于同一个地形形态作各种不同的立体显示，更好地研究地形的空间形态。

  3. 通视分析：通视分析有着广泛的应用背景。典型的例子是观察哨所的设定、森林中火灾监测点的设定、无线发射塔的设定等。通视问题可以分为五类：a）已知一个或一组观察点，找出某一地形的可见区域；b）欲观察到某一区域的全部地形表面，计算最少观察点数量；c）在观察点数量一定的前提下，计算能获得的最大观察区域；d）以最小代价建造观察塔，要求全部区域可见；e）在给定建造代价的前提下，求最大可见区。根据问题输出维数的不同，通视可分为点的通视，线的通视和面的通视。

  4. 流域特征地貌提取与地形自动分割：是进行流域空间模拟的基础技术。主要包括两个方面：a）流域地貌形态结构定义，定义能反映流域结构的特征地貌，建立格网DEM对应的微地貌特征；b）特征地貌自动提取和地形自动分割算法。

  5. 计算地形属性：DEM派生的地形属性数据可以分为单要素属性和复合属性二种。前者可由高程数据直接计算得到，如坡度因子，坡向。后者是由几个单要素属性按一定关系组合成的复合指标，用于描述某种过程的空间变化，这种组合关系通常是经验关系，也可以使用简化的自然过程机理模型。

通视分析是指以某一点为观察点，研究某一区域通视情况的地形分析。利用DEM判断地形上任意两点之间是否可以互相可见的技术方法，分为视线通视分析和可视域分析，前者判断任意两点之间或者多点之间能否通视，后者对于给定的观察点，分析观察所覆盖的区域。其中可视域是从一个或者多个观察的可以看见的地表范围。可视域分析是在栅格数据数据集上，对于给定的一个观察点，基于一定的相对高度，查找给定的范围内观察点所能通视覆盖的区域，也就是给定点的通视区域范围，分析结果是得到一个栅格数据集。

在什么情况下需要用矢量与栅格的互换，如何实现

当进行空间线性拓扑关系分析以及网络分析，单个实体的定义和操作和对成图精度质量要求高时，具体当进行城市分区规划、详细规划、土地管理等时用矢量数据格式，这时需把栅格 数据转换为矢量数据。 

当进行缓冲区分析、与遥感影像或其它图像匹配处理、数据共享时，具体当进行大范围小比例尺的环境、农林等区域问题研究时，以及建立地理实体模型和进行 DEM 地形分析时，需把矢量转换为栅格。

矢量转换为栅格的实现方法有：1. 内部点扩散算法；2. 复数积分法；3. 扫描法；4. 射线法；5. 边界代数算法。

具体步骤是：1. 选择合理的栅格尺寸；2. 点的栅格化；3. 线的栅格化；4. 面的填充。 

栅格转换为矢量的过程：1. 边界提取；2. 二值化；3. 细化；4. 边界跟踪；5. 拓扑关系生成；6. 去除多余点和边界光滑处理。 

栅格数据结构与矢量数据结构：理解、比较、转换

栅格数据结构

栅格结构是以规则的阵列来表示空间地物或现象分布的数据组织，组织中的每个数据表示地物或现象的非几何属性特征。

栅格结构的显著特点:属性明显，定位隐含，即数据直接记录属性的指针或数据本身，而所在位置则根据行列号转换为相应的坐标。

栅格数据的编码方法:直接栅格编码，就是将栅格数据看作一个数据矩阵，逐行（或逐列）逐个记录代码；压缩编码，包括链码（弗里曼链码）比较适合存储图形数据；游程长度编码通过记录行或列上相邻若干属性相同点的代码来实现；块码是有成长度编码扩展到二维的情况，采用方形区域为记录单元；四叉树编码是最有效的栅格数据压缩编码方法之一，还能提高图形操作效率，具有可变的分辨率。

矢量数据结构

矢量数据结构是通过记录坐标的方式尽可能精确地表示点、线和多边形等地理实体，坐标空间设为连续，允许任意位置、长度和面积的精确定义。

矢量结构的显著特点:定位明显，属性隐含。

矢量数据的编码方法:

对于点实体和线实体，直接记录空间信息和属性信息；

对于多边形地物，有坐标序列法、树状索引编码法和拓扑结构编码法。坐标序列法是由多边形边界的x， y坐标对集合及说明信息组成，是最简单的一种多边形矢量编码法，文件结构简单，但多边形边界被存储两次产生数据冗余，而且缺少邻域信息；树状索引编码法是将所有边界点进行数字化，顺序存储坐标对，由点索引与边界线号相联系，以线索引与各多边形相联系，形成树状索引结构，消除了相邻多边形边界数据冗余问题；拓扑结构编码法是通过建立一个完整的拓扑关系结构，彻底解决邻域和岛状信息处理问题的方法，但增加了算法的复杂性和数据库的大小。

矢量栅格数据的比较

矢量数据的优缺点:

优点为数据结构紧凑、冗余度低，有利于网络和检索分析，图形显示质量好、精度高；图形数据和属性数据的恢复、更新、综合都能实现，面向目标，不仅能表达属性，而且能方便的记录每个目标的具体属性信息。

缺点为数据结构复杂，多边形叠加分析比较困难。数学模拟比较复杂，技术复杂。

栅格数据的优缺点:

优点为数据结构简单了，空间数据的叠置和组合方便，各类空间分析很容易进行，数学模拟方便。

缺点为数据量大，投影转换比较复杂。用大像元减少数据量时，精度和信息量受损，地图输出不美观，难以简历网络连接关系，投影变换比较费时。

两者比较:

栅格数据操作总的来说容易实现，矢量数据操作则比较复杂；

栅格结构是矢量结构在某种程度上的一种近似，对于同一地物达到于矢量数据相同的精度需要更大量的数据；在坐标位置搜索、计算多边形形状面积等方面栅格结构更为有效，而且易于遥感相结合，易于信息共享；矢量结构对于拓扑关系的搜索则更为高效，网络信息只有用矢量才能完全描述，而且精度较高。对于地理信息系统软件来说，两者共存，各自发挥优势是十分有效的。

矢量栅格相互转换算法

矢量转栅格:内部点扩散法，即由多边形内部种子点向周围邻点扩散，直至到达各边界为止；复数积分算法，即由待判别点对多边形的封闭边界计算复数积分，来判断两者关系；射线算法和扫描算法，即由图外某点向待判点引射线，通过射线与多边形边界交点数来判断内外关系；边界代数算法，是一种基于积分思想的矢量转栅格算法，适合于记录拓扑关系的多边形矢量数据转换，方法是由多边形边界上某点开始，顺时针搜索边界线，上行时边界左侧具有相同行坐标的栅格减去某值，下行时边界左侧所有栅格点加上该值，边界搜索完毕之后即完成多边形的转换。

栅格转矢量:即是提取具有相同编号的栅格集合表示的多边形区域的边界和边界的拓扑关系，并表示成矢量格式边界线的过程。步骤包括:多边形边界提取，即使用高通滤波将栅格图像二值化；边界线追踪，即对每个弧段由一个节点向另一个节点搜索；拓扑关系生成和去处多余点及曲线圆滑

矢量数据结构与栅格数据结构的转换算法

对于点状实体，每个实体仅由一个坐标对表示，其矢量结构和栅格结构的相互转换基本上只是坐标精度变换问题。线实体的矢量结构由一系列坐标对表示，在变为栅格结构时，除 把序列中坐标对变为栅格行列坐标外，还需根据栅格精度要求，在坐标点之间插满一系列栅格点，这可以由两点式直线方程得到。线实体由栅格结构变为矢量结构与将多边形边界 表示为矢量结构相似，因此以下重点讨论多边形的矢量结构与栅格结构相互转换。

1.矢量向栅格转换

矢量格式向栅格格式转换又称为多边形填充，就是在矢量表示的多边形边界内部的所有栅格点上赋以相应的多边形编码，从而形成栅格数据阵列。几种主要的算法描述如下：

a）内部点扩散算法：由每个多边形一个内部点（种子点）开始，向其八个方向的邻点扩散，判断各个新加入点是否在多边形边界上，如果是边界上，则该新加入点不作为种子点，否则把非边界点的邻点作为新的种子点与原有种子点一起进行新的扩散运算，并将该种子点赋以该多边形的编号。重复上述过程直到所有种子点填满该多边形并遇到边界停止为止 。 r，则该待判点属于此多边形，赋以多边形编号，否则在此多边形外部，不属于该多边形。

b）复数积分算法：对全部栅格阵列逐个栅格单元地判断该栅格归属的多边形编 码，判别方法是由待判点对每个多边形的封闭边界计算复数积分，对某个多边形，如果积分值为2

c）射线算法和扫描算法：射线算法可逐点判断数据栅格点在某多边形之外或 在多边形内，由待判点向图外某点引射线，判断该射线与某多边形所有边界相交的总次数，如相交偶数次，则待判点在该多边形外部，如为奇数次，则待判点在该多边形内部。要 注意的是：射线与多边形边界相交时，有一些特殊情况会影响交点的个数，必须予以排除。

d）边界代数算法：它适合于记录拓扑关系的多边形矢量数据转换为栅格结构。若多 边形编号为a，初始化的栅格阵列各栅格值为零，以栅格行列为参考坐标轴，由多边形边界上某点开始顺时针搜索边界线，当边界上行时，位于该边界左侧的具有相同行坐标的所 有栅格被减去a；当边界下行时，该边界左边所有栅格点加一个值a，边界搜索完毕则完成了多边形的转换。

2.栅格格式向矢量格式的转换

多边形栅格格式向矢量格式转换就是提取以相同的编号的栅格集合表示的多边形区域的边界和边界的拓扑关系，并表示由多个小直线段组成的矢量格式边界线的过程。栅格格式向矢量格式转换通常包括以下四个基本步骤：

a）多边形边界提取：采用高通滤波将栅格图像二值化或以特殊值标识边界点；

b）边界线追踪：对每个边界弧段由一个结点向另一个结点搜索，通常对每个已知边界点需沿除了进入方向的其他7个方向搜索下一个边界点，直到连成边界弧段；

c）拓扑关系生成：对于矢量表示的边界弧段数据，判断其与原图上各多边形的空间关系，以形成完整的拓扑结构并建立与属性数据的联系；

d）去除多余点及曲线圆滑：由于 搜索是逐个栅格进行的，必须去除由此造成的多余点记录，以减少数据冗余；搜索结果，曲线由于栅格精度的限制可能不够圆滑，需采用一定的插补算法进行光滑处理，常用的算 法有：线形迭代法；分段三次多项式插值法；正轴抛物线平均加权法；斜轴抛物线平均加权法；样条函数插值法。

5. 什么是数字高程模型DEM

数字高程模型（Digital Elevation Model）是一个有着均匀空间间隔的高程数据的数组。

数字高程模型（Digital Elevation Model)，简称DEM，是通过有限的地形高程数据实现对地面地形的数字化模拟（即地形表面形态的数字化表达），它是用一组有序数值阵列形式表示地面高程的一种实体地面模型，是数字地形模型（Digital Terrain Model，简称DTM）的一个分支，其它各种地形特征值均可由此派生。

　　DEM分辨率是DEM刻画地形精确程度的一个重要指标，同时也是决定其使用范围的一个主要的影响因素。DEM的分辨率是指DEM最小的单元格的长度。因为DEM是离散的数据，所以（X，Y）坐标其实都是一个一个的小方格，每个小方格上标识出其高程。这个小方格的长度就是DEM的分辨率。分辨率数值越小，分辨率就越高，刻画的地形程度就越精确，同时数据量也呈几何级数增长。

由于DEM描述的是地面高程信息，它在测绘、水文、气象、地貌、地质、土壤、工程建设、通讯、军事等国民经济和国防建设以及人文和自然科学领域有着广泛的应用。如在工程建设上，可用于如土方量计算、通视分析等；在防洪减灾方面，DEM是进行水文分析如汇水区分析、水系网络分析、降雨分析、蓄洪计算、淹没分析等的基础；在无线通讯上，可用于蜂窝电话的基站分析等等。

DEM是一种数据类型，有很多常用格式，比如TIF、hgt、img等，那么在使用的时候，需要注意自己的目标应用软件支持什么格式。不过好的一点是，DEM的所有格式都可以相互转换。

对于DEM并没有一个标准的定义。按照DEM的结构，即其数据组织方式，可以分为基于点、基于线单元和基于面单元的DEM。其中基于点的DEM实质上就是离散采样点的集合；基于线单元的DEM主要是基于等高线DEM；基于面单元的DEM又分为基于规则格网（如正方形格网、正六边形格网等）的DEM和基于不规则格网的DEM。DEM可以以栅格、矢量、TIN和grid的形式表示。

在ArcGIS中，一般将基于TIN的DEM简记为TIN，将基于规则格网DEM记为DEM，将与矢量数据相对应的数据记为栅格。并且，这三种数据之间可相互转换。

（4）空间数据使用中的误差：主要包括两个方面：一是对数据的解释过程，一是缺少文档，这样往往导致数据用户对数据的随意性使用而使误差扩散。数据质量控制是个复杂的过程，要控制数据质量应从数据质量产生和扩散的所有过程和环节入手，分别用一定的方法减少误差。

数据质量

数据质量包括：

  1. 准确性，即一个记录值与它的真实值之间的接近程度；

  2. 精度，即对现象描述的详细程度；

  3. 空间分辨率，即两个可测量数值之间最小的可辨识的差异；

  4. 比例尺，即地图上一个记录的距离和它所表现的真实距离之间的一个比值；

  5. 误差，即一个所记录的测量和它的事实之间的差异；

  6. 不确定性，包括空间位置的不确定性、属性不确定性和数据不完整性等。

GIS数据质量包含如下五个方面：

①位置精度：如数学基础、平面精度、高程精度等，用以描述几何数据的质量。

②属性精度：如要素分类的正确性、属性编码的正确性、注记的正确性等，用以反映属性数据的质量。

③逻辑一致性：如多边形的闭合精度、结点匹配精度、拓扑关系的正确性等。

④完备性：如数据分类的完备性、实体类型的完备性、属性数据的完备性、注记的完整性等。

⑤现势性：如数据的采集时间、数据的更新时间等。

数据质量控制方法

空间数据质量控制常见的方法有：

（1）传统的手工方法：将数字化数据与数据源进行比较，图形部分的检查包括目视方法、绘制到透明图上与原图叠加比较，属性部分的检查采用与原属性逐个对比或其他比较方法。

（2）元数据方法：数据集的元数据中包含了大量的有关数据质量的信息，通过它可以检查数据质量，同时元数据也记录了数据处理过程中质量的变化，通过跟踪元数据可以了解数据质 量的状况和变化。

（3）地理相关法：用空间数据的地理特征要素自身的相关性来分析数据的质量。

Differential correction uses correction factors available from a reference (base) station to reduce noise errors of data collected by a GPS receiver. 1. 仿射变换包括哪三个步骤，在操作上

  1. update the x and y coordinates of selected control points to real-world coordinates.

  2. run an affine transformation on the control points to real-world coordinates.

  3. use the estimated coefficients and the transformation equations to compute the x and y coordinates of map features in the digitized map or pixels in the image.

  4. 将选定的控制点的x和y坐标更新为现实世界的坐标。

  5. 对控制点进行仿射变换，使之成为现实世界的坐标。

Step 2：在控制点上运行仿射变换，并检查RMS Error（均方根误差）；

控制点用于估算仿射变换系数。此系数用于计算地图、影像的x， y坐标。

因为从图像到地图的转换所创建的新图像没有像素值，所以必须进行重采样，用一个值或从原始图像中得出的值来填充新图像的每个像素。

因为变换得到的新地图没有像元值，需要重采样将源图像的像元值填入新地图。

4. 描述栅格数据三种常用的重采样方法

Nearest neighbor， bilinear interpolation， and cubic convolution. The nearest neighbor sampling method fills each pixel of the new image from the original image. The bilinear interpolation method uses the average of the four nearest pixel values from three linear interpolations. And the cubic convolution method uses the average of the 16 nearest pixel values from five cubic polynomial interpolations.

邻近点插值法、双线性插值法、三次卷积插值法、五次多项式插值法。

Computational efficiency is the main advantage of using an elevation raster for terrain analysis. The simple raster data structure makes it relatively easy to perform neighborhood operations on the elevation raster.

计算效率是使用高程栅格进行地形分析的主要优势。简单的栅格数据结构使得对高程栅格进行近邻操作变得相对容易。

7. 使用TIN进行地形制图和分析有什么优点

The main advantage of using a TIN for terrain mapping and analysis is the flexibility with the input data sources. One can construct a TIN using DEMs， contour lines， GPS data， LINDAR data， and survey data. One can also modify the initial TIN with point， line， and area features. Additionally， a TIN is an excellent data model for 3-D display.

使用TIN进行地形测绘和分析的主要优点是输入数据来源的灵活性。人们可以用DEMs、等高线、GPS数据、LINDAR数据和测量数据来构建TIN。人们还可以用点、线和区域特征来修改初始TIN。此外，TIN是一个很好的3-D显示的数据模型。

第14章 视域和流域分析

1. 视域分析所需的输入数据的两种类型

The first input to a viewshed analysis is a point layer that contains one or more viewpoints. If a line layer is used， the viewpoints are points (vertices and nodes) that make up the linear features. The second input is a DEM (elevation raster) or a TIN， which represents the land surface.

A filled DEM or elevation raster is void of depressions. Unless these depressions are removed from an elevation raster， they become areas of internal drainage， thus distorting the drainage pattern.

第15章 空间插值

1. 什么是空间插值

Spatial interpolation is the process of using points with known values to estimate values at other points.

空间插值是使用具有已知值的点来估计其他点的值的过程。

空间插值则是一种通过这些离散的空间数据计算未知空间数据的方法. 它是基于"地理学第一定律"的基本假设: 空间位置上越靠近的点， 具有相似特征值的可能性越大， 而距离远的点， 其具有相似特征值的可能性越小。它通常用于将离散点的测量数据转换为连续数据表面， 以便于比较其他空间现象的分布情况。

2. 空间插值需要何种输入数据

Spatial interpolation requires two basic inputs: control points and an interpolation method. Control points are points with known values， which provide the data necessary for the development of an interpolator. An interpolation method or an interpolator is typically a mathematical equation for estimating unknown values.

空间插值需要两个基本输入：控制点和插值方法。控制点是具有已知值的点，可提供生成插值器所需的数据。插值方法或插值器通常是用于估计未知值的数学方程

3. 说明整体拟合插值和局部拟合插值的区别

A global interpolation method uses every known point available to estimate an unknown value， whereas a local interpolation method uses a sample of known points that are close to the point to be estimated to estimate an unknown value.

全局插值方法使用每个可用的已知点来估计未知值，而局部插值方法使用靠近要估计的点的已知点的样本来估计未知值。

整体插值方法通常不直接用于空间插值，而是用来检测总趋势和不同于总趋势的最大偏离部分，即剩余部分，在去除了宏观趋势后，可用剩余残差来进行局部插值。

整体插值方法通常使用方差分析和回归方程等标准的统计方法，计算比较简单。

全局多项式插值法适用于在数据集中创建平滑表面及标识长期趋势。然而，在地球科学中，除了长期趋势之外，感兴趣的变量通常还具有短程变化。当数据集显示出短程变化时，局部多项式插值法地图可捕获这种变化。

局部多项式插值法对邻域距离很敏感，较小的搜索邻域可能会在预测表面内创建空区域。因此，可以在生成输出图层之前预览表面。

4. 精确插值法与非精确插值法的区别是什么

An exact interpolation predicts a value at the point location that is the same as its known value， whereas an inexact interpolation， or approximate interpolation， method predicts a value at the point location that differs from its known value.

精确插值预测点位置处的值与其已知值相同，而不精确插值或近似插值方法预测点位置处的值与其已知值不同。

对于某个数据已知的点，精确插值法在该点位置的估算值与该点已知值相同。也就是，精确插值所生成的面通过所有控制点，而非精确插值或叫做近似插值，估算的点值与该点已知值不同。

5. 什么是泰森多边形

The interpolation method of Thiessen polygons is based on the assumption that any point within a polygon is closer to the polygon's known point than any other known points.

泰森多边形的插值方法基于这样一个假设，即多边形内的任何点都比任何其他已知点更接近多边形的已知点。

泰森多边形可用于定性分析、统计分析、近邻分析等。

泰森多边形的特性是：

1，每个泰森多边形内仅含有一个离散点数据。

2，泰森多边形内的点到相应离散点的距离最近。

3，位于泰森多边形边上的点到其两边的离散点的距离相等。

6. 说明半方差如何用于对数据集中的空间相关性进行定量化

The semivariance can be used to quantify the spatial dependence in a data set because， if spatial dependence does exist in the data set， known points that are close to each other are expected to have larger semivariances.

半方差可用于量化数据集中的空间相关性，因为如果数据集中确实存在空间相关性，则预计彼此靠近的已知点将具有更大的半方差。

7. 半方差函数必须先用数学模型拟合，然后才能用于克里格法。为什么

A semivariogram must be fitted with a mathematical model so that fitted semivariogram can be used for estimating the semivariance at any given distance.

半方差函数必须与数学模型拟合，以便拟合半方差函数可用于估计任何给定距离的半方差。

拟合半方差函数可用于估计任何给定距离的半方差。

8. IDW和克里金估算法都使用权重来估计未知值。描述两种插值方法在赋予权重方面的差异

Weights used in kriging involve not only the semivariances between the point to be estimated and the known points but also those between the known points. Weights used in the IDW method involve only weights applicable to the point to be estimated and the known points.

克里金法中使用的权重不仅涉及待估计点与已知点之间的半方差，还涉及已知点之间的半方差。IDW方法中使用的权重只涉及适用于待估计点和已知点的权重。

9. 描述普通克里格与克里金之间的主要区别

Ordinary kriging focuses on the spatially correlated component and uses the fitted semivariogram directly for interpolation. Universal kriging， on the other hand， assumes that the spatial variation in z values has a drift or a trend in addition to the spatial correlation between the sample points.

普通克里金法侧重于空间相关分量，并直接使用拟合半变异函数进行插值。另一方面，通用克里金法假设除了样本点之间的空间相关性之外，z 值的空间变化还具有漂移或趋势。

10. 均方根误差通常用于选择最佳插值方法。RMS量测的是什么

RMS measures the average deviation between the known value and the estimated value of the control points.

RMS 测量控制点的已知值和估计值之间的平均偏差。

11. 解释如何使用验证技术来比较不同的插值方法

The validation technique compares the interpolation methods by first dividing known points into two samples: one sample for developing the model for each interpolation method to be compared and the other sample for testing the accuracy of the models.

验证技术通过首先将已知点分成两个样本来比较插值方法：一个样本用于为要比较的每种插值方法开发模型，另一个样本用于测试模型的准确性。

第18章 GIS模型与建模

1. 描述描述模型和规则模型之间的区别

A descriptive model describes the existing conditions of geospatial data， whereas a prescriptive model offers a prediction of what the conditions could be or should be.

描述性模型描述了地理空间数据的现有条件，而规范性模型提供了条件可能或应该是什么的预测。

2. 静态模型与动态模型有什么不同

A dynamic model emphasizes the changes of geospatial data and the interactions between variables， whereas a static model deals with the state of geospatial data at a given time.

动态模型强调地理空间数据的变化和变量之间的相互作用，而静态模型处理给定时间地理空间数据的状态。

3. 描述建模过程中涉及的基本步骤

  1. define the goals of the mode.

  2. break down the model into elements and to define the properties of each element and the interactions between the elements.

  3. implement and calibrate the model.

  4. validate the model under conditions that are different from those used in the calibration phase.

  5. 定义模式的目标。

  6. 将模型分解为元素并定义每个元素的属性以及元素之间的交互。

  7. 实施和校准模型。

  8. 在不同于校准阶段使用的条件下验证模型。

4. 索引模型与二值模型有什么不同

An index model produces for each unit area an index value rather than a simple yes or no， as in a binary model.

索引模型为每个单位面积生成一个索引值，而不是像二进制模型那样简单的是或否。

5. 许多指数模型使用加权线性组合方法计算指标值。解释使用加权线性综合法的步骤

  1. evaluate the relative importance of each criterion against other criteria.

  2. standardize data for each criterion.

  3. calculate the index value for each unit area by summing the weighted criterion values and dividing the sum by the total of the weights.

  4. 根据其他标准评估每个标准的相对重要性。

  5. 标准化每个标准的数据。

  6. 通过将加权标准值相加并除以权重的总和来计算每个单位面积的指标值。

6. 加权线性组合法有哪些优缺点

Weighted linear combination cannot deal with interdependence between factors， does not allow trade-off weighting， and is inflexible in terms of the derivation of the index value and the standardization of criterion values.

加权线性组合不能处理因素之间的相互依赖，不允许权衡加权，在指标值的推导和标准值的标准化方面缺乏灵活性。

7. 如何区分局部回归模型与回归模型

A local regression model uses the information for each known point to derive a local model. The model's parameters can therefore vary in space， providing a basis to explore spatial nonstationarity (i.e.， the relationship between variables varies over space)， as opposed to stationarity (i.e. the relationship between variables remains the same over space) assumed in a global regression model.

局部回归模型使用每个已知点的信息来推导局部模型。

因此，模型的参数可以在空间中变化，为探索空间非平稳性提供基础（即变量之间的关系随空间变化），这与全局回归模型中假设的平稳性相反（即变量之间的关系在空间上保持不变）。

8. 什么是环境模型

An environmental model integrates existing knowledge about an environmental process in the real world into a set of relationships and equations for quantifying the process.

环境模型将现实世界中有关环境过程的现有知识整合到一组关系和方程中，用于量化过程。
